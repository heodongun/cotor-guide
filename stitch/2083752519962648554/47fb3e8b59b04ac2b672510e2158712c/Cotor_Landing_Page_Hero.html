menu_book Core Concepts

Understanding the architecture of Cotor is essential for building scalable multi-agent systems. The system revolves around two primary entities: the Master Agent and Worker Nodes.

Architecture Overview

The Master Agent acts as the orchestrator, receiving complex tasks and decomposing them into sub-tasks. It then routes these sub-tasks to specialized Worker Nodes based on their defined capabilities and current load.

Diagram showing Master Agent orchestrating multiple Worker Nodes.

The Master Agent

The Master Agent evaluates the overall goal and utilizes a strategic planning model (e.g., gpt-4-turbo) to create an execution graph. It handles:

  • Task Decomposition
  • Dependency Management
  • Error Recovery and Retry Logic
  • Final Synthesis of results

Worker Nodes

Worker Nodes are execution units that focus on specific domains. A worker could be specialized in web scraping, data analysis, or executing external APIs. They operate concurrently to speed up the overall task completion.

Configuration Example

Below is a typical YAML configuration for defining a Cotor system.

cotor.yaml 
version: '2.0'
system:
  name: "Research-Cluster"
  strategy: "hierarchical"
  master:
    model: "gpt-4-turbo"
    capabilities: [delegation, synthesis]
agents:
  - name: "Data-Gatherer"
    role: "Web Scraping & API Interaction"
    tools: [browser, serp_api]
    concurrency: 5
  - name: "Analyst"
    role: "Data Processing & Summarization"
    dependencies: ["Data-Gatherer"]
